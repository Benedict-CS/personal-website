# 後台與 Blog / Notes 編輯與閱讀 UX 優化建議

> 針對後台、部落格編輯/閱讀、筆記編輯/閱讀的體驗與方便性整理。

---

## 一、Blog 編輯體驗（`/dashboard/posts/[id]`）

### 已具備
- MDEditor 即時預覽、工具列、圖片上傳、版本歷史、標籤/分類/發布日等欄位。

### 建議優化

| 項目 | 說明 | 優先級 |
|------|------|--------|
| **鍵盤快捷鍵存檔** | `Ctrl+S` / `Cmd+S` 觸發存檔，避免誤觸瀏覽器另存頁面。 | 高 |
| **預覽按鈕** | 一鍵在新分頁開啟「讀者看到的樣子」：已發布開 `/blog/[slug]`，草稿開 `/dashboard/notes/[slug]`。 | 高 |
| **存檔後不跳轉** | 存檔成功後可選「留在編輯頁」或「返回列表」，減少來回切換。 | 中 |
| **編輯器高度 / 全螢幕** | 內容區可拉高或切換「專心模式」（只顯示標題+內容），減少捲動。 | 中 |
| **自動儲存草稿** | 每 N 秒或失焦時自動 PATCH 內容到後端，降低未存檔遺失風險。 | 中（實作較重） |
| **從列表區分 Blog / Note** | Posts 列表用篩選或 Tab 區分「已發布」與「草稿(筆記)」，快速找到要編輯的項目。 | 低 |

---

## 二、Blog 閱讀體驗（前台 `/blog/[slug]`）

### 已具備
- ToC、閱讀進度條、上一篇/下一篇、分享、標籤、Edit 按鈕（登入時）。

### 建議優化

| 項目 | 說明 | 優先級 |
|------|------|--------|
| **字體 / 行距可調** | 偏好設定或文章頁開關：字體大小、行距，存於 localStorage。 | 低 |
| **列印樣式** | `@media print` 隱藏導航、進度條，只留標題與內文，方便列印/PDF。 | 低 |

---

## 三、Notes 編輯體驗

### 現況
- 筆記 = 未發布的 Post，編輯入口：Notes 列表的「Edit」或筆記閱讀頁的「Edit」→ `/dashboard/posts/[id]`（與 Blog 共用同一編輯頁）。

### 建議優化

| 項目 | 說明 | 優先級 |
|------|------|--------|
| **列表直接進編輯** | 已有一鍵「Edit」到 `/dashboard/posts/[id]`，可保留；若希望「點標題就進編輯」可改為預設進編輯，另提供「View」進閱讀。 | 中（依個人習慣） |
| **與 Blog 編輯一致** | 同一編輯頁已支援分類、標籤、預覽；確保「Preview」在草稿時開筆記閱讀頁即可。 | 高（見上節） |
| **快速改分類** | Notes 列表已有內聯改分類，可保留；若常拖曳排序，再考慮拖曳改順序/分類。 | 低 |

---

## 四、Notes 閱讀體驗（`/dashboard/notes/[slug]`）

### 已具備
- 標題、標籤、分類、最後更新、上一篇/下一篇（同分類）、Markdown 渲染、checkbox 可勾選並寫回 DB。

### 建議優化

| 項目 | 說明 | 優先級 |
|------|------|--------|
| **目錄 (ToC)** | 與 Blog 相同：側邊或頂部顯示 H2/H3，點擊捲動到對應標題，長文筆記更好導覽。 | 高 |
| **閱讀進度條** | 與 Blog 相同：頂部一條進度條，捲動時顯示閱讀進度。 | 高 |
| **版面與 Blog 一致** | 主文 + 右側 ToC 的兩欄版面，視覺與 Blog 閱讀一致，減少認知負擔。 | 中 |
| **鍵盤導覽** | 上一篇/下一篇支援鍵盤（如 ← / →），方便連續閱讀。 | 低 |

## 五、後台整體方便性

| 項目 | 說明 | 優先級 |
|------|------|--------|
| **側邊欄當前頁標示** | 當前路徑（如 `/dashboard/notes`）對應的 nav 項目高亮或加粗，方便辨識所在區塊。 | 高 |
| **全域快捷鍵** | 例如 `G then P` 進 Posts、`G then N` 進 Notes（可選）。 | 低 |
| **未存檔提示** | 編輯頁若有未存檔變更，離開前（beforeunload 或 Next 導覽）提示「有未儲存變更」。 | 中 |
| **Posts 列表篩選** | 依狀態（已發布/草稿）、標籤、分類篩選，方便在大量文章中找文。 | 中 |

---

## 六、實作優先順序建議

1. **先做、成本低**
   - 編輯頁：`Ctrl+S` 存檔、預覽按鈕（依發布狀態開 Blog 或 Note 閱讀頁）。
   - Notes 閱讀頁：加上 ToC、閱讀進度條，並可改為兩欄（主文 + ToC）。
   - Dashboard 側邊欄：當前頁高亮。

2. **其次**
   - 存檔後可選「留在編輯頁」。
   - 編輯頁「未存檔就離開」的提示。
   - Posts 列表簡單篩選（已發布/草稿）。

3. **可選**
   - 自動儲存草稿、編輯器全螢幕/專心模式、鍵盤導覽上一篇/下一篇、列印樣式。

以上依「編輯與閱讀的流暢度、方便性」整理，可依實際使用習慣再微調優先級。
